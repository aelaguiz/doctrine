# `system-prompts-and-models`

This source is a mixed archive of production prompts, tool schemas, and role definitions from several coding agents.
The high-signal material is where a prompt combines a concrete role with an explicit operating envelope: exact tool names and parameters, read-only planning, mode classification, and narrow safety guardrails.

Use this source to pressure Doctrine around:

- exact tool contracts instead of vague capability prose
- split behavior by mode or role, especially planning versus execution
- hard limits on file creation, docs creation, or system changes
- concise classifiers that choose a branch from user intent

What to ignore:

- product branding and release framing
- commentary about vendor ecosystems
- any decorative prose that does not change behavior
