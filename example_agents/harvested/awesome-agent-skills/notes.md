# `awesome-agent-skills`

This source is a catalog, not a skill body.
That makes it useful in a different way: it shows how a skill ecosystem is organized, named, grouped, and quality-controlled.

Use this source to pressure Doctrine around:

- catalog and registry surfaces for skills
- metadata quality standards such as concise descriptions, relative paths, and scoped tools
- taxonomy for broad capability families without leaking vendor-specific details into the core language
- clean separation between an index of skills and the actual skill bodies

What to ignore:

- the sheer volume of linked skill names
- brand-heavy section headings
- any linked skill title unless it helps define a durable taxonomy or packaging rule
